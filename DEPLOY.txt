StreamEngine — deploy from GitHub (minimal clicks)
===================================================

What is already IN THE REPO (netlify.toml — Netlify UI overrides the same key names):
  • Build: npm ci && npm run build, publish dist
  • /api/* on Netlify proxies to the same base URL as VITE_API_ORIGIN (generated at build; see scripts/emit-netlify-redirects.mjs)
  • SPA catch-all for client routing
  • Defaults: VITE_SPOTIFY_CLIENT_ID, VITE_APP_URL, VITE_SUPABASE_URL (rltejlyhvhlivskbezkx)
  • render.yaml → API on Render: Spotify client id, SUPABASE_URL, CORS to your Netlify site

Netlify → Environment variables (typical production set):
  VITE_API_ORIGIN — REQUIRED for production API calls: your live Render web service root (https://<name>.onrender.com). Open it in a browser: GET /health must return JSON. Plain “Not Found” means nothing is deployed at that hostname yet — deploy the API on Render first, then paste the real URL here (Build + Run scope).
  VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY, VITE_STRIPE_PUBLISHABLE_KEY, VITE_SPOTIFY_CLIENT_ID
  Spotify: the browser only reads VITE_SPOTIFY_CLIENT_ID (or the built-in default in src/lib/env.js). A variable named SPOTIFY_CLIENT_ID without the VITE_ prefix is ignored by Vite — duplicate the same Client ID into VITE_SPOTIFY_CLIENT_ID if you set it in the UI.
  + VITE_APP_URL if it differs from netlify.toml.
  After edits: Deploys → Trigger deploy.

IMPORTANT — VITE_API_ORIGIN is a URL, not a key:
  No trailing slash, no path. Use the exact URL Render shows for your Web Service (not a guess).
  Do NOT paste SPOTIFY_CLIENT_SECRET or STRIPE_SECRET_KEY here — those belong only on Render (server).
  Never put STRIPE_SECRET_KEY in Netlify with a VITE_ prefix (it would ship to the browser).

You still must add SECRETS once on the API host (they must not be committed to Git):
  On Render → your API service → Environment:
    SPOTIFY_CLIENT_SECRET, STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET,
    SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, (+ optional OPENAI_API_KEY, STRIPE_SUBSCRIPTION_PRICE_ID)

If the site shows “demo mode” or Supabase never connects: confirm VITE_SUPABASE_ANON_KEY is set on Netlify and redeploy.
Optional: npx netlify-cli env:import .env (from a machine that has a complete .env).

If your Netlify URL is not streamenginev1.netlify.app, edit netlify.toml (VITE_APP_URL) and render.yaml (APP_URL + CORS_ALLOW_ORIGINS).

If your Render service URL differs from what you first guessed, set VITE_API_ORIGIN on Netlify to that URL and redeploy (the Netlify /api proxy is rebuilt from this variable each build).

Supabase — Google / Apple OAuth (manual dashboard checklist)
------------------------------------------------------------
After code deploy, complete these in Supabase (rltejlyhvhlivskbezkx or your project):

  1. Authentication → Providers → enable Google; add Web client ID + secret from Google Cloud Console.
  2. Authentication → Providers → enable Sign in with Apple if you use it; add Service ID / key / bundle IDs per Apple’s flow.
  3. Authentication → URL Configuration → Site URL = your live app origin (e.g. https://streamenginev1.netlify.app).
  4. Redirect URLs: add https://streamenginev1.netlify.app/ and http://localhost:5173/ (or your dev port) so OAuth can return to the app.
  5. Confirm provider credentials are saved; “provider is not enabled” in the UI means the toggle or keys are still missing in Supabase.

The app calls supabase.auth.signInWithOAuth with providers `google` and `apple` and uses VITE_APP_URL when it matches the current origin for redirectTo; otherwise it uses window.location.origin.

Exclusive lane (guests): POST /api/create-exclusive-guest-checkout — no userId; Stripe metadata billing_kind=exclusive_guest. Webhook/sync skips Supabase credits.

Render — Web Service settings (if you create the API by hand)
--------------------------------------------------------------
  Root directory: (empty = repo root)
  Build command:   npm ci
  Start command:   node server/index.js
  Health check:    /health
  Node:            20.x (repo has .node-version and render.yaml NODE_VERSION; avoids accidental Node 25+)

Render API — HTTP routes (Express)
-----------------------------------
Health:
  GET  /health              — JSON { ok, service, ts } (Render health check)
  GET  /api/health          — same payload under /api

Spotify:
  GET  /api/spotify/track?url=…     — track metadata (intl + spotify:track:… URLs supported)
  GET  /api/spotify/playlist?url=…
  GET  /api/spotify/health

Stripe / billing:
  POST /api/create-checkout
  POST /api/billing/create-checkout
  POST /api/create-subscription-checkout
  POST /api/sync-checkout
  POST /api/billing/webhook          — raw body; set endpoint URL in Stripe dashboard
  POST /api/billing/create-payment-intent
  POST /api/validate-discount
  POST /api/billing/validate-discount
  GET  /api/billing/health

CORS: set APP_URL (and/or comma-separated origins) plus CORS_ALLOW_ORIGINS on Render so browser calls from your Netlify origin(s) are allowed. Local dev origins (localhost, 127.0.0.1, LAN) are allowed automatically.
