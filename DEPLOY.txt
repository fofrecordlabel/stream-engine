StreamEngine — deploy from GitHub (minimal clicks)
===================================================

What is already IN THE REPO (no Netlify env screen needed for these):
  • netlify.toml  → build command, SPA routing, VITE_SPOTIFY_CLIENT_ID, VITE_APP_URL, VITE_API_ORIGIN
  • render.yaml     → API on Render with Spotify client id + CORS to your Netlify site

You still must add SECRETS once (they must not be committed to Git):
  On Render → streamengine-api → Environment:
    SPOTIFY_CLIENT_SECRET, STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET,
    SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, (+ optional OPENAI_API_KEY, STRIPE_SUBSCRIPTION_PRICE_ID)

Netlify build also needs your public Supabase + Stripe publishable keys (safe in repo but unique per project).
If the Netlify build fails with “demo mode” or missing Supabase, add these in ONE of two ways:
  A) Netlify → Site → Environment variables → add VITE_SUPABASE_URL, VITE_SUPABASE_ANON_KEY, VITE_STRIPE_PUBLISHABLE_KEY
  B) One terminal (with your local .env):  npx netlify-cli env:import .env

If your Netlify URL is not streamenginev1.netlify.app, edit netlify.toml (VITE_APP_URL) and render.yaml (APP_URL + CORS_ALLOW_ORIGINS).

If your Render API name is not streamengine-api, change the Render service name and set VITE_API_ORIGIN in netlify.toml to match (https://<name>.onrender.com, no trailing slash).
